# Local AI-Powered SLA Analytics & Incident Orchestration Engine

A production-grade, privacy-first automation workflow built in n8n that processes daily contact center support telemetry, calculates trailing operational performance benchmarks, leverages a **locally hosted Large Language Model (LLM)** for executive analysis, and orchestrates live notifications alongside dynamic Markdown dashboards.

---

## 🚀 Architectural Overview

This system is engineered to replace expensive, cloud-dependent analytical layers with a completely private, low-latency infrastructure stack. It executes on a scheduled daily cron job to ingest raw ticketing statistics, calculate trailing 7-day metrics, detect critical infrastructure outages, and gracefully update cross-functional business platforms.

### 🗺️ System Topology & Data Flow
1. **Ingestion & Processing:** Automatically parses daily ticket snapshots and references historical data backlogs on local disk.
2. **Analytical Computation:** Computes performance metrics and flags infrastructure outage thresholds using native JavaScript data blocks.
3. **Local Intelligence Layer:** Streams processed telemetry payload to a self-hosted **Local AI Engine** running on-premises to extract executive-level insights without corporate data ever leaving the private perimeter.
4. **Multi-Channel Orchestration:**
   - **Discord API Hub:** Broadcasts automated, color-coded health status alerts and granular AI briefings directly to operations teams.
   - **Obsidian Vault (KM Layer):** Seamlessly writes to a local knowledge base, constructing an optimized executive dashboard featuring self-rendering UI states depending on system health.

---

## 🛠️ Tech Stack & Core Engineering

- **Workflow Orchestration:** n8n (Advanced node layouts & custom binary stream handling)
- **Data Manipulation:** Vanilla JavaScript ES6 (Custom regex string extraction, data buffers, and dynamic Base64 encoding)
- **Generative AI Platform:** Local AI Engine (Self-hosted Open-Source LLM Architecture)
- **Notification Bus:** Discord Webhooks API
- **Knowledge Management Interface:** Obsidian Markdown Core Engine

---

## 💎 Key Enterprise-Grade Highlights

### 1. Dynamic UI/UX State Rendering
The pipeline evaluates raw telemetry programmatically within custom JavaScript code blocks. If a system outage is flagged, the downstream storage adapter conditionally shifts UI layouts, injecting native Obsidian callout wrappers (`> [!success]` vs `> [!danger]`) and converting metric numbers into isolated code block tokens to guarantee immediate scannability for engineering managers.

### 2. Privacy-First & Zero-Cost AI Operations
By bypassing proprietary cloud LLM APIs (like OpenAI) in favor of an on-premises engine, the architecture guarantees **zero variable operational API costs** and ensures strict internal enterprise data compliance guidelines are satisfied.

### 3. Modular Future Roadmap Architecture
The dataset is persisted as clean, structured markdown pairs (`tickets:: value`). This explicitly positions the project for next-phase deployment scales:
- **Obsidian Dataview Framework:** Enabling immediate offline relational database queries and metric charting.
- **Local RAG Integration:** Preparing a vectorized knowledge base for local context retrieval.
- **Incident Escalation:** Extensible branching directly into Jira, ServiceNow, or PagerDuty APIs.

## ⚙️ How To Deploy Locally
1. Prerequisites
n8n Instance: Running locally or via Docker.

Local AI Instance: Running an open-source LLM model endpoint locally.

Obsidian App: A local directory setup to receive incoming file writes.

2. Installation Steps
Download the workflow.json blueprint file included in this repository.

Open your n8n workspace, click your top-right menu icon, and click Import.

Select the workflow.json file.

Update your local directory credentials inside the Archive Report to Company Records node to point to your target directory path.

Click Execute Workflow to run your first automated loop cycle.
